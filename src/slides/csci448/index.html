How is a web app different from a mobile app?

A mobile app is downloaded once from an app store, stored on the phone, and periodically recieves updates. Generally written in Java (Android) or Objective-C/Swift(iOS).

A mobile web app is served over HTTP, and used within the browser. Written in HTML, CSS, and JavaScript.

It's basically the difference between the Facebook app on your phone, and mobile.facebook.com.

It's faster and easier to update a mobile web app, and you only have to develop a single app that varies slightly between browsers. However, mobile apps get access to all of the native, speedy UI and computational power that the OS has.

First things first

Go back to your index.html and replace it with: 

					<h1>Adventure time quotes!</h1>
<p>this is where our quote goes</p>

This defines two things:

  • <h1> defines a "Heading 1" for our app. We'll just have this be the title of our app.
  • <p> defines a "paragraph" tag. This will be where our app displays my favorite quote!

At the end of index.html, let's include a <script> tag. 

						<script>
	// our code is going to go here
</script>

The <script> tag tells the browser, "There's JavaScript in here!"

JavaScript is what we use to manipulate content on the page.

// defines a JavaScript comment.

Let's include some quotes that we like as an Array of Strings in our <script> tag. 

		<script>

Here, we're defining a new (untyped) variable called quotes.

We're setting quotes equal to an array (using square brackets, []) of strings (using quotation marks "")

Don't forget commas after each element!

Now, let's find the <p> tag, and replace its contents with an element selected at random from our array of quotes. 

					<script>
	var quotes = ["Mathematical!",

	document.getElementsByTagName('p')[0].innerHTML =

</script>

whoa whoa whoa

Okay, let's work through this line: 

					document.getElementsByTagName('p')[0].innerHTML =
	quotes[Math.floor(Math.random() * quotes.length)]

document is an object that the browser provides to us. It's got lots of different methods and variables.

getElementsByTagName is a method of document that takes a string as an argument (in this case, the tag name that we're looking for) and returns an array of HTML elements (objects).

more document-ation on document (ha, get it?): https://developer.mozilla.org/en-US/docs/Web/API/Document 

					document.getElementsByTagName('p')[0].innerHTML =
	quotes[Math.floor(Math.random() * quotes.length)]

[0] is a reference to the first element (the element at index 0) of the array produced by document.getElementsByTagName('p'). In this case, that array only has a length of 1, but if there were more <p> tags on the page, it would be larger.

innerHTML is a reference to the actual content of the element that we've referenced in getElementsByTagName. It's got lots more variables and methods, like children and hasAttribute().

Here's the element-ary documentation on Elements: https://developer.mozilla.org/en-US/docs/Web/API/Element 

					document.getElementsByTagName('p')[0].innerHTML =
	quotes[Math.floor(Math.random() * quotes.length)]

Finally, we pick a quote at random, by multiplying a random number (from 0 to 1) by the length of the array, and then flooring the number to get an integer.

In the previous session, we did some front end web development - computation and logic only occurred on the client, or the page within the browser.

This session, we're going to have our client call a server and retrieve a quote. This is back end web development - the server is going to do the computation (picking a quote at random) and then send the information back to the client.

(quick demo of what we're doing today)

How is this different from what we did before?

Before, every time we sent the HTML over the wire, we'd include all of our quotes in that HTML, resulting in a larger page.

This time, we're going to store the quotes on the server, and have a (smaller) page just request one at random every time.

Note that this method won't work on Firebase anymore - if you're interested in simple Node.js server hosting, check out Heroku or Modulus.

You know how we can run JavaScript in the browser? We can also run JavaScript in Node.js!

Things look a little different between the two: In the browser, you get some builtin variables/objects, like document and window

In Node, you get a whole different bunch of builtin variables/objects, like fs and require

Node.js is built on top of the V8 JavaScript engine: http://thibaultlaurens.github.io/javascript/2013/04/29/how-the-v8-engine-works/

To get a Node.js REPL loop, just run node on the command line. (You can play around with it later)

Project setup

Run: npm init

Next, we need to set up our Node.js script to "listen" for HTTP requests on our machine.

We're going to use a library called Express (http://expressjs.com/).

Installing Express

On the command line, run npm install express --save

This installs the express library locally to our project.

You may notice that a node_modules folder has been created - feel free to explore it, but we won't dive deep into it this session.

Inside of your index.js file, enter: 

					var express = require('express');
var app = express();

app.listen(3000, function () {
	console.log("Listening on port 3000...");
})

Now, try running node index.js. What happens? What happens if we try to navigate to http://localhost:3000/?

Spoiler alert: Nothing!

Whoops, we're currently listening for HTTP requests on port 3000, but we're not actually responding to them. Let's fix that!

Make a call to the app.get method in your index.js: 

					var express = require('express');
var app = express();

app.get("/", function(request, response) {
	response.sendFile(__dirname + "/index.html");
})

app.listen(3000, function() {
	console.log("Listening on port 3000...");
})

Don't forget to restart your script! It's still running the old Node.js script. You'll need to restart it every time you make changes to your index.js file.

Then navigate back to http://localhost:3000/...

Hooray!

It should show you the index.html file you created earlier!

Next step: AJAX

We need to have our page (index.html) call our server (our Node.js script) to retrieve the latest quote (using AJAX), and insert the quote into our page.

AJAX stands for Asynchronous JavaScript and XML, but that's not really something that you need to know.

General AJAX documentation (no libraries here!): https://developer.mozilla.org/en-US/docs/AJAX

Back on index.html, let's include jQuery at the top of the page. 

					<script src="https://ajax.googleapis.com/ajax/libs/jquery/2.2.2/jquery.min.js"></script>

jQuery is a library that wraps a lot of the standard document and element methods, and makes it easier to manipulate the page. Generally you don't need it for most manipulation, though.

jQuery documentation: http://api.jquery.com/

Aside: The browser exposes a native method for making HTTP requests, called XMLHttpRequest, but it's kind of a pain to use. jQuery wraps it for us, provides some other functionality, and makes it a little easier to use.

More info on XMLHttpRequest: https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest

In the script that's at the bottom of our page, replace the contents with: 

	$.ajax("/random_quote").done(function (data) {
	document.getElementsByTagName('p')[0].innerHTML = data;
})

This makes an HTTP request to http://localhost:3000/random_quote, gets a response back, and replaces the contents of the p tag with the data.

Literally everything you ever wanted to know about jQuery's ajax method: http://api.jquery.com/jquery.ajax/

Restart your script, reload the page, and head back on over to http://localhost:3000/...
